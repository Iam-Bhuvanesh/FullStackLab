SyncFlow AI is an enterprise-grade web platform engineered to demonstrate modern HTML5 standards. It seamlessly integrates advanced inputs, interactive drag-and-drop workspace cards, and persistent storage mechanisms into a high-contrast responsive interface.

Form & Storage Objectives

To provide accessible, user-friendly data entry with built-in client validation, editable notes, and instant state retrieval.

Interactive Drag Objectives

To implement standard HTML5 Drag & Drop API event listeners (`dragstart`, `dragover`, `drop`) for intuitive user interactions.

Semantic HTML5

About This Web Application

Application Architecture & Design Rationale

This application demonstrates standard HTML5 semantic architecture. By utilizing elements like <header>, <nav>, <main>, <section>, <article>, <aside>, and <footer>, the document ensures maximum accessibility, search engine optimization (SEO), and clean code maintainability.

Key HTML5 features showcased in this lab project include:

  • Advanced Input Types: Native HTML5 controls for Email, Tel (with pattern validation), Date, Time, and Number (with min/max constraints).
  • Interactive Content Editing: Rich text notes using contenteditable="true" with browser spellcheck="true" integration.
  • Drag and Drop API: Visual workspace modules that can be dragged into dynamic target drop zones using JavaScript drag events.
  • Web Storage API: Dual storage architecture using localStorage (permanent user state) and sessionStorage (temporary session tokens).

Help & Documentation Manual

1. How to Test HTML5 Form Validation

Fill out the fields under Registration Form. Try leaving required fields empty or putting invalid patterns into Email/Phone/Age to test browser native validation warnings.

2. How to Test Drag & Drop

Click and hold any module card inside Drag Source Palette, move your cursor over Active Workspace Dropzone or Staging Archive, and release your mouse button.

3. How to Test Web Storage

Enter values in Local Storage or Session Storage and click Save. Click Retrieve Data to see the stored JSON object in the inspection terminal. Refresh the browser to verify Local Storage persistence!
